# AIoT Assistant 答辩说明

## 项目定位

AIoT Assistant 是一个面向物联网专业课程的 Android 智能助手。它把 AI 问答、本地学习记录、模型调用统计、多模态附件和本地智能体任务整合在一个 App 中，适合用于课程学习、实验辅助和课堂演示。

## 演示流程

1. 启动 App，先展示短时专业欢迎页，突出 AIoT Assistant 的专业和智能调性。
2. 首次启动继续展示 Welcome Page，说明 AIoT 问答、学习记录和本地任务能力。
3. 进入聊天页，展示新建会话、历史会话抽屉和本地消息保存。
4. 打开设置页，选择模型预设或新增自定义配置，输入 API Key，保存为当前模型并测试连接。
5. 回到聊天页，在顶部模型切换条切换配置，发送物联网问题，展示 loading、错误处理、AI 回复和系统通知。
6. 演示复制消息、失败回复重试、重新生成最后一条问题和代码块可读展示。
7. 添加拍照、文件或录音附件，展示多模态附件卡片和文本预览。
8. 打开统计页，展示会话、消息、模型调用、Token/字符估算、7 日趋势、网络/API 状态。
9. 打开 Claw 智能体页，演示短信确认、系统拍照、WorkManager 下载和任务日志。
10. 打开设置页，演示主题切换、调试信息导出和清空历史。

## 核心技术点

- Compose 单 Activity 架构：`MainActivity` 作为入口，`AppNavigation` 管理页面路由。
- 启动欢迎页：使用 Compose Canvas 绘制拓扑网络、微光节点和 AI 标识，短时展示后自动进入 Onboarding 或主界面。
- Material 3 UI：统一页面容器、状态标签、卡片组件和浅色/深色主题。
- Room 持久化：会话、消息、模型统计、智能体任务分表保存。
- DataStore 设置：保存多套模型配置、当前激活模型、主题模式和 Onboarding 状态。
- OpenAI 兼容协议：通过 OkHttp 请求 `/v1/chat/completions`，适配 DeepSeek、通义、MiMo、OpenAI 和自定义服务。
- 流式输出：聊天回复使用 OpenAI 兼容 SSE，模型分片返回时同步更新同一条 assistant 消息。
- 聊天体验增强：消息复制使用系统剪贴板，重新生成和失败重试复用 Repository 的 AI 回复请求流程。
- 多模态附件：使用 FileProvider、系统相机、文件选择器和 MediaRecorder 完成附件采集。
- Vision 请求升级：图片附件可按模型配置转换为 OpenAI 兼容 `image_url` content part，不支持或读取失败时自动文本降级。
- 统计监测：使用 Room Flow、DataStore 和网络监听实时更新页面数据，并展示当前模型配置和最近 API 失败原因。
- WorkManager：下载服务以后台任务执行，支持进度更新和完成通知。
- Android 权限：相机、录音、通知、短信都通过系统权限模型控制。

## 数据设计

| 表 | 作用 |
| --- | --- |
| `conversations` | 保存会话标题、摘要、模型、消息数量和更新时间 |
| `messages` | 保存用户/AI 消息、附件 JSON、loading 状态和 Token 估算 |
| `model_usage` | 保存模型调用次数、Token/字符估算和调用时间 |
| `agent_tasks` | 保存 Claw 模式短信、拍照、下载任务状态 |

## 安全与隐私说明

- API Key 不写入源码，只在本机 DataStore 中保存。
- 多模型配置支持 DeepSeek、通义、MiMo、OpenAI 和自定义服务；聊天页可以快速切换当前模型。
- 聊天历史、附件元数据和统计数据都存储在本机 Room 数据库。
- 短信任务不能自动静默执行，必须用户输入、授权并二次确认。
- 拍照任务通过系统相机完成，不绕过用户交互。
- 设置页提供清空历史会话功能，便于演示后清理数据。

## 现场演示注意事项

- 演示真实 AI 回复前，需要在设置页填入真实 API Key。
- 如果现场网络不稳定，可以先展示本地会话、设置、统计和 Claw 模式，再测试 API。
- 短信演示建议使用测试号码和真机；没有 SIM 卡时可展示权限、二次确认和失败日志。
- 下载演示建议准备一个稳定的小文件 URL。
- 拍照演示需要设备上有可用系统相机。
- 图片理解演示需要选择支持 vision 的模型，并在模型配置中开启“支持图片输入”。
