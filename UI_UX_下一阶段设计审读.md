# lot UI/UX 下一阶段设计审读

审读日期：2026-06-17
更新日期：2026-06-18

## 产品定位

`lot` 不是课堂工具，也不是物联网专用工具，而是面向普通用户的 Android AI 智能助手。物联网只是作业题目背景，不应成为 App 的产品边界。它的气质应接近“通用 AI 助手 + 本地能力工作台”：能长期承载对话、资料整理、写作润色、附件采集、模型配置、统计监测和本地智能体任务。

参考 `ui-ux-pro-max` 给出的设计基准，本项目更适合采用：

- Material 3 + 扁平化移动端工具风。
- 单色/中性色底盘，保留蓝色作为 AI 主色，绿色/青色作为在线、成功和本地能力状态色，橙色只用于关键提醒或高价值 CTA。
- 信息密度高但可扫读，避免营销页式的大留白和装饰卡片。
- 所有触控目标至少 44-48dp，优先使用清晰图标与语义标签。

## 2026-06-18 已落实进度

- App 对外定位已修正为通用 AI 智能助手；底层 `SYSTEM_PROMPT`、欢迎页、聊天空状态、统计洞察、设置页版本标识和 README 均已同步。
- 已新增 `LotDesignTokens.kt`，抽出 spacing、radius、motion 和常用语义色/图表色。
- 聊天页已完成多行 Composer、能力条、AI/Claw segmented control、通用 Prompt Deck 和 DeepSeek 风格历史抽屉分组。
- 设置页已加入模型配置向导和当前模型概览，降低首次配置成本。
- 统计页已加入自动洞察卡，并统一部分图表 palette。
- `Claw` 已从隐藏能力变为底部导航顶层入口，新增本地能力面板，展示短信、拍照、下载和 App-Leap。
- 已完成 `assembleDebug`、`testDebugUnitTest`、`connectedDebugAndroidTest` 回归验证。

## 当前 UI 优点

- 主导航已经调整为“对话 / 智能体 / 统计 / 设置”，四个顶层页面都直接服务 AI 助手核心工作流。
- `AppPage`、`AppSectionCard`、`StatusPill` 已经形成基础设计组件，便于下一阶段抽出统一设计 token。
- 聊天页已有 DeepSeek 风格的历史抽屉雏形，支持搜索、重命名、删除和 Claw 日志切换。
- 统计页已具备图表、模型调用、网络状态和任务统计，满足原始评分项中“统计与监测页面”的主体要求。
- 欢迎页使用 Canvas 绘制节点网络，有轻量 AI 科技感，是可继续强化 lot 品牌记忆的好入口。

## 优先级 1：建立 lot 设计系统

### 1. 抽出设计 token

当前硬编码颜色分散在多个页面，例如：

- `ChatScreen.kt` 中 Claw 使用 `Color(0xFF39FF88)`。
- `StatsScreen.kt` 图表网格和 palette 使用多组硬编码颜色。
- `AgentScreen.kt` 仍有终端绿、雷达蓝等独立色值。

建议新增 `ui/theme/LotDesignTokens.kt`：

- `LotSpacing`: `xs=4.dp`, `sm=8.dp`, `md=12.dp`, `lg=16.dp`, `xl=24.dp`
- `LotRadius`: `sm=6.dp`, `md=8.dp`, `lg=12.dp`
- `LotMotion`: `fast=120`, `normal=220`, `slow=320`
- `LotSemanticColor`: `ai`, `assistant`, `claw`, `warning`, `success`, `chartPalette`

收益：下一阶段美化时不用在每个页面改硬编码颜色，暗色模式也更容易统一。

### 2. 品牌视觉统一为“lot”

当前启动页文字已改为 `lot`，LogoMark 也已经使用 `lot`。下一步建议继续把图标系统和启动页视觉统一到小写品牌名：

- 主标识：小写 `lot`，其中 `o` 可以做成轻量节点环或对话焦点。
- 启动页节点拓扑保留，但避免暗示某个垂直行业。
- App icon 下一阶段也应同步，避免桌面显示名是 lot、图标仍像模板应用。

## 优先级 2：聊天页重塑为“AI 助理工作台”

### 1. 顶部区域做成紧凑状态栏

当前聊天页顶部有历史按钮、会话摘要、Claw 开关、模型切换、新建按钮、思考开关。功能完整，但信息像“横向排满控件”。

建议改成两层但视觉更稳定：

- 第一层：`lot` / 当前会话标题 / 历史入口图标 / 新建图标。
- 第二层：模型胶囊、思考状态、Claw 状态合并成“能力条”。
- Claw 开关改成 segmented control：`AI` / `Claw`，比 Switch 更像模式切换，也更容易被用户理解。

### 2. 输入区改为“Composer Bar”

当前附件入口是 `+` 文本，发送按钮为文字。建议：

- 附件入口改为明确图标按钮：相机、文件、麦克风放进底部弹出菜单，`+` 只作为更多入口。
- 发送按钮改为圆形箭头/纸飞机图标，Claw 模式改为闪电/执行图标。
- 输入框支持多行，最多 4 行；长问题在当前单行输入里可读性不足。
- 待发送附件卡片缩成横向 preview strip，多个附件未来扩展时可自然横向滚动。

### 3. 消息气泡增强“专业助手感”

当前用户、AI、Claw 气泡已区分颜色，但每条消息顶部都显示角色和时间，会让长对话显得偏重。

建议：

- 同角色连续消息只在第一条显示角色标签，时间移到轻量 meta 行。
- AI 消息增加左侧细色条或小标识，用户消息保持右侧对齐。
- 代码块增加复制按钮、语言标签、横向滚动；这对代码解释、配置示例和长文本资料都很关键。
- 失败消息视觉上使用错误边框 + “重试”主按钮，而不是普通 TextButton。

### 4. 空会话做成通用 Prompt Deck

空状态已从物联网专用问题调整为通用 AI 助手场景。下一步可以继续优化为更有生活感和生产力感的 6 个问题卡片：

- “帮我润色一段自我介绍”
- “把这段知识点讲明白”
- “制定一周复习计划”
- “把这段资料整理成摘要”
- “写一封礼貌的邮件”
- “帮我比较两个方案”

视觉上分成“写作 / 学习 / 规划 / 总结 / 生成 / 分析”六类，小图标 + 短标题 + 一行提示。这样更像普通用户可立即使用的 AI 助理，而不是测试样例。

## 优先级 3：历史会话列表向 DeepSeek 靠近

原始要求明确“历史会话列表样式类似 DeepSeek 的会话列表”。当前抽屉已有基础，但下一步可以更像 DeepSeek：

- 会话按日期分组：今天、昨天、近 7 天、更早。
- 列表项只显示标题，摘要在长按或展开时显示，减少密度。
- 选中会话使用左侧 3dp 活跃条，而不是只靠背景色。
- 删除和重命名放到 trailing overflow，不常驻两个图标，降低视觉噪音。
- 搜索框置顶 sticky，滚动历史时保留。

## 优先级 4：统计页从“有图表”升级为“可洞察”

当前图表可视化已经丰富，但有三个问题：硬编码 palette、移动端图表可读性、图表语义不足。

建议：

- 顶部增加一句自动洞察，例如“今天模型调用比昨天多 3 次，主要来自 mimo-v2.5”。
- 图表卡片统一结构：标题、时间范围、主指标、图表、图例、洞察。
- 颜色不要只表示模型，增加直接标签或短编号，避免用户只靠颜色记忆。
- Canvas 图表增加空数据、少数据、大数据三种视觉状态。
- 图表点击后的 insight 文案做成底部浮层或卡片，不只是一行文本，便于用户复盘使用情况。

适合下一阶段新增的创意图表：

- “模型能耗雷达”：调用次数、prompt、completion、失败次数、最近活跃。
- “对话热力日历”：显示最近 7/30 天的对话活跃度。
- “Claw 任务漏斗”：发起、授权、执行、完成、失败。

## 优先级 5：设置页降低认知负担

设置页功能强，但模型编辑弹窗字段多，用户第一次配置时会有压力。

建议：

- 新增配置流程拆成两步：选择服务商预设 -> 填 Key 和模型。
- 能力开关折叠为“高级能力”，默认只显示图片、思考、转写的摘要。
- API Key 输入增加“本地保存，不会出现在调试导出中”的固定 helper text。
- 连接测试结果不要只是一行文本，改为状态卡：成功 / 401 / 网络失败 / Base URL 错误，并给恢复建议。
- 当前模型卡增加“立即去对话”的入口，减少配置后找不到下一步。

## 优先级 6：动效与反馈

原始评分里动效占比高，当前主要动效集中在启动页和消息加载。下一阶段建议增加有意义的动效：

- 会话切换：消息列表轻微 crossfade，保留滚动位置。
- 新消息到达：只对新气泡做 160ms fade + slide-up。
- 模型回复中：把线性进度条改为“思考脉冲点 + 当前模型名”。
- 附件加入：附件卡片从输入区上方展开，移除时收起。
- Claw 模式：切换时输入栏边框和执行按钮颜色过渡，强化“模式改变”。

所有动效应控制在 120-300ms，不阻塞点击，并预留 reduced motion 支持。

## 优先级 7：可访问性和触控

建议下一阶段统一检查：

- 所有 IconButton 都有 contentDescription，目前部分按钮已做，但历史动作、图表、Canvas 交互还需要补齐。
- IconButton 触控区域不低于 44/48dp；设置页配置操作按钮当前 36dp，建议视觉图标 18dp、触控外框 48dp。
- 统计图表不能只靠颜色区分，应增加标签、图例和文字摘要。
- 长文本不要一律 maxLines + ellipsis；关键错误和 API 失败原因应可展开。
- 深色模式下检查图表网格线、边框、次级文字对比度。

## 建议的下一阶段拆分

1. 设计 token 与品牌基线：抽出 token、替换硬编码颜色、统一 lot 标识。
2. 聊天页 Composer Bar：图标化附件/发送、多行输入、附件 preview strip。
3. DeepSeek 风格历史抽屉：日期分组、overflow 操作、sticky 搜索。
4. 统计页洞察卡：自动洞察、图表状态、图例和点击反馈重做。
5. 设置页配置向导：服务商预设、两步配置、高级能力折叠。
6. 动效强化：新消息、会话切换、附件加入、Claw 模式切换。
7. 可访问性验收：触控区域、contentDescription、深浅色对比、动态字体。

## 验收建议

- 小屏手机、普通手机、横屏各走一遍聊天、设置、统计。
- 打开深色模式检查气泡、图表、状态 pill、弹窗。
- 用 1.3x 或更大系统字体检查输入栏、底部导航、模型配置卡是否溢出。
- 用模拟器重跑 `connectedDebugAndroidTest`，确保 UI 文案或层级调整没有破坏测试。
- UI 截图纳入报告：启动页、空会话、长对话、历史抽屉、统计图表、设置模型编辑。
